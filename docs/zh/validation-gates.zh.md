# 验证门禁

本页面记录了目前用于验证 ProofForge 的可运行门禁，并将其与计划中但尚未实现的门禁区分开来。它反映了实际的脚本、根 `justfile` 配方以及 `.github/workflows/ci.yml`；它不会添加或编辑 CI 作业。

## 可选外部工具

当前的 CI 安装了 `just` 1.48.0、Foundry stable 和 `solc` 0.8.30。本地机器可能没有 `just`、`solc`、`cast`、`forge`、`psyup`、`dargo`、`sbpf`、`surfpool`、Solana CLI、Node 或 npm。缺少 `just` 会阻塞本地命令目录，但不会阻塞直接的脚本执行。缺少 EVM 工具会阻塞 EVM 工具链门控，但不会阻塞 `lake build`。缺少 Psy 工具仅会阻塞 Psy 冒烟测试的 Dargo 部分；源代码生成和黄金差异（golden diff）在每个脚本退出前仍会运行。缺少 Solana 工具会阻塞 Solana 汇编/运行时冒烟测试，但不会阻塞 Lean 构建或目标注册表检查。

所有基于 Dargo 的 Psy 冒烟测试脚本还会在 `src/main.psy` 记录生成的 Dargo 包源代码副本，并在 `proof-forge-artifact.json` 记录生成的 `Dargo.toml` 包清单。元数据验证器检查包源代码路径、字节大小、SHA-256 哈希以及与生成的源文件的哈希一致性。它还会检查清单路径、字节大小、SHA-256 哈希、`[package]` 部分、`type = "bin"` 和 `[dependencies]` 部分。包生成通过 `scripts/psy/write-dargo-package.py` 共享，因此每个冒烟测试都使用相同的源代码副本和清单布局。
